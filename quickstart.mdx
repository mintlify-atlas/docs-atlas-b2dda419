Get your OpenATS development environment set up and create your first job posting in under 10 minutes.

## Prerequisites

Before you begin, ensure you have the following installed:

- **Node.js** (version 18 or higher) - [Download here](https://nodejs.org/)
- **Git** - [Download here](https://git-scm.com/)
- **PostgreSQL database** - We recommend [Neon](https://neon.tech) (free tier, no local install needed)
- **Code editor** - VS Code recommended

Verify your installation:

```bash
node --version
git --version
```

## Installation

<Steps>
  <Step title="Clone the repository">
    Clone the OpenATS repository to your local machine:

    ```bash
    git clone https://github.com/chamals3n4/OpenATS.git
    cd OpenATS
    ```

    <Check>Verify you're in the OpenATS directory by running `ls` - you should see `api` and `web` folders</Check>
  </Step>

  <Step title="Install pnpm">
    OpenATS uses pnpm as its package manager. Install it globally:

    <CodeGroup>
    ```bash npm
    npm install -g pnpm
    ```

    ```bash yarn
    npm install -g pnpm
    ```

    ```bash pnpm
    # If you already have pnpm installed
    pnpm --version
    ```
    </CodeGroup>

    <Check>Verify installation with `pnpm --version`</Check>
  </Step>

  <Step title="Install dependencies">
    Install dependencies for both the frontend and backend:

    **Frontend (web):**
    ```bash
    cd web
    pnpm install
    ```

    **Backend (api):**
    ```bash
    cd ../api
    pnpm install
    ```

    <Check>Wait for all dependencies to install - this may take a few minutes</Check>
  </Step>

  <Step title="Configure environment variables">
    Set up environment variables for both applications.

    **Frontend environment:**
    ```bash
    cd web
    cp .env.example .env
    ```

    Edit `web/.env` and configure your authentication settings (WSO2 Asgardeo).

    **Backend environment:**
    ```bash
    cd ../api
    cp .env.example .env
    ```

    Edit `api/.env` with your configuration:

    ```bash .env
    # Database (Required)
    DATABASE_URL=postgresql://user:password@host:port/database

    # Cloudflare R2 Storage (Optional - for file uploads)
    R2_ENDPOINT=https://your-account-id.r2.cloudflarestorage.com
    R2_ACCESS_KEY_ID=your_access_key_id
    R2_SECRET_ACCESS_KEY=your_secret_access_key
    R2_BUCKET_NAME=your_bucket_name
    R2_PUBLIC_URL=https://your-public-url.com

    # Email (Optional - for notifications)
    RESEND_API_KEY=re_your_api_key
    RESEND_FROM_EMAIL=noreply@yourdomain.com
    ```

    <Note>
    **Database URL Format:**
    - For Neon: `postgresql://user:password@host/database?sslmode=require`
    - For local PostgreSQL: `postgresql://localhost:5432/openats`
    </Note>

    <Check>Ensure your DATABASE_URL is correctly configured - this is required for the next steps</Check>
  </Step>

  <Step title="Set up the database">
    Run database migrations and seed data from the `api` directory:

    **Generate and run migrations:**
    ```bash
    pnpm drizzle-kit generate
    pnpm drizzle-kit migrate
    ```

    **Seed the database:**
    ```bash
    pnpm tsx src/db/seed.ts
    ```

    This creates all necessary tables and inserts default hiring pipeline stages.

    <Check>Verify migrations completed successfully - you should see confirmation messages</Check>
  </Step>

  <Step title="Start the development servers">
    Run both the frontend and backend servers in separate terminal windows.

    **Terminal 1 - Backend API:**
    ```bash
    cd api
    pnpm dev
    ```

    The API will start on `http://localhost:5000`

    **Terminal 2 - Frontend Web:**
    ```bash
    cd web
    pnpm dev
    ```

    The web app will start on `http://localhost:3000`

    <Check>Open `http://localhost:3000` in your browser - you should see the OpenATS interface</Check>
  </Step>
</Steps>

## Creating Your First Job Posting

Now that OpenATS is running, let's create your first job posting.

<Steps>
  <Step title="Access the application">
    Navigate to `http://localhost:3000` in your browser and sign in with your configured authentication provider.
  </Step>

  <Step title="Navigate to Jobs">
    From the dashboard, click on the "Jobs" section in the navigation menu.
  </Step>

  <Step title="Create a new job">
    Click the "Create Job" or "New Job Posting" button.
  </Step>

  <Step title="Fill in job details">
    Enter the following information:
    
    - **Job Title** (e.g., "Senior Software Engineer")
    - **Department** (e.g., "Engineering")
    - **Location** (e.g., "Remote" or "San Francisco, CA")
    - **Employment Type** (Full-time, Part-time, Contract)
    - **Job Description** (Responsibilities, requirements, etc.)
    - **Salary Range** (Optional)

    <Tip>Use the rich text editor to format your job description with bullet points and sections</Tip>
  </Step>

  <Step title="Configure hiring pipeline">
    The default hiring pipeline stages are automatically applied:
    
    - Applied
    - Screening
    - Interview
    - Offer
    - Hired

    You can customize these stages later in the settings.
  </Step>

  <Step title="Publish the job">
    Click "Save" or "Publish" to create your job posting. It will now be visible to applicants and your hiring team.

    <Check>Your first job posting is live! You can now start receiving applications.</Check>
  </Step>
</Steps>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Database connection failed">
    **Problem:** Error connecting to PostgreSQL database.

    **Solutions:**
    - Verify your `DATABASE_URL` in `api/.env` is correct
    - Ensure your database server is running
    - For Neon, check that you've added `?sslmode=require` to the connection string
    - Test the connection using a PostgreSQL client
  </Accordion>

  <Accordion title="Port already in use">
    **Problem:** Error: `EADDRINUSE: address already in use :::3000` or `:::5000`

    **Solutions:**
    - Kill the process using the port:
      ```bash
      # On macOS/Linux
      lsof -ti:3000 | xargs kill -9
      lsof -ti:5000 | xargs kill -9
      
      # On Windows
      netstat -ano | findstr :3000
      taskkill /PID <PID> /F
      ```
    - Or change the port in your configuration
  </Accordion>

  <Accordion title="Module not found errors">
    **Problem:** Error: `Cannot find module 'X'`

    **Solutions:**
    - Delete `node_modules` and reinstall:
      ```bash
      rm -rf node_modules
      pnpm install
      ```
    - Ensure you're using the correct Node.js version (18+)
    - Clear pnpm cache: `pnpm store prune`
  </Accordion>

  <Accordion title="Migration errors">
    **Problem:** Database migration fails or tables not created.

    **Solutions:**
    - Ensure your database is empty or drop existing tables
    - Run migrations in order:
      ```bash
      pnpm drizzle-kit generate
      pnpm drizzle-kit migrate
      ```
    - Check database permissions - your user needs CREATE TABLE privileges
  </Accordion>

  <Accordion title="Authentication not working">
    **Problem:** Cannot sign in or authentication redirects fail.

    **Solutions:**
    - Verify WSO2 Asgardeo configuration in `web/.env`
    - Check that callback URLs are correctly configured in your Asgardeo application
    - Ensure your client ID and client secret are correct
    - Clear browser cookies and try again
  </Accordion>

  <Accordion title="File upload errors">
    **Problem:** Cannot upload resumes or attachments.

    **Solutions:**
    - Verify R2 configuration in `api/.env` (if using Cloudflare R2)
    - Check that your R2 bucket exists and is accessible
    - Verify access key credentials are correct
    - Ensure your bucket has the correct CORS policy
  </Accordion>
</AccordionGroup>
